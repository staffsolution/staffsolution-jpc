# 各情報に理由と働く人への影響を入れる（フィードバック）

2025-11-25

**フィードバック動画：** <a href="https://www.loom.com/share/ff71a27fefa64d548087bab4ec925f6e" target="_blank">Loomで見る</a>

---

## 相談内容

自社リサーチの情報を整理した状態でのフィードバック。投資額・スープの仕込み・メニュー・提供時間・駐車場などの情報に対し、理由と「働く人への影響」を加える指示。

## 全体の評価

読むだけで「めちゃくちゃいいお店」と思えるくらい情報が揃ってきている。あと一歩として、各情報に「なぜそうなのか（理由）」と「それが働く人にどう影響するか（翻訳）」を入れることでさらに伝わる内容になる。

## 各セクションのフィードバック

### 投資額（個人店の4倍・全国チェーンの1.2倍）

- 理由：なぜ一般のお店は投資を抑えるのかが入ると差が明確になる
- 理由：なぜ一般のお店はスープを店外で製造し、保存料・化学調味料・油を多く使うのかが分かると「ここが違う」が伝わる
- 自社：スープを店内で仕込み、できたてを提供しているから美味しい。その差が読み手に伝わる形にする

### 清潔な環境（クリーニングを外注しない理由）

- 清潔な環境を保つためにかけているコスト・手間の理由を入れる
- 一般店では清潔感にそこまで投資しない場合がある → 自社は清潔さにも投資している → 働く人が気持ちよく働ける環境につながる

### 駐車場の広さ

- 働く人がスタッフとして車を止める際に広くて止めやすい → 通勤・業務のストレスが少ない → 「働きやすい」という翻訳をここに入れる

### こだわりの美味しさ・楽しさ

- おいしさ（スープ・自分好みにカスタマイズできる）と楽しさ（イベント・オープンキッチンでのライブ感）をここでまとめる
- 「ここで働いていることが誇れる」という軸でまとめると強い
- 一般は味だけ、ここはおいしさ＋楽しさという差を出す

### メニュー数が絞られている

- 覚えることが少ない → 注文を受ける際に混乱しない → すぐに覚えられる → 成長が早い
- 「働きやすい」という文脈で整理する

### 提供時間が早い（30秒）

- なぜ30秒で提供できるのかの理由を入れる
- それが働く人にどう影響するか：流れるように動けてストレスが少ない / 覚えることが少ない / ポジションごとに動けて成長が早い
- この内容を「働きやすさ」のセクションにまとめる

## 整理の方向性

各情報に「理由（なぜそうなのか）」と「働く人への翻訳（それがどう影響するか）」の2点を入れる。事実だけ並べると伝わりにくい。「だからあなたにとってこういうメリットがある」まで書く。

---

## やりがちなミス

- 投資額・設備・メニュー数など「事実」を並べるだけで、「なぜそうなのか」と「働く人にどう関係するか」が抜ける
- 清潔さ・設備に関する情報を「お店の自慢」として書き、求職者目線の翻訳をしていない
- 提供時間が早いことを書いても「だから何？」が伝わらない

## 他の案件でも使えるパターン

- 自社の強みを書くときは「事実 → なぜそうなのか → 働く人（または求職者）への影響」の3点セットで書く
- 「○秒・○倍・○メニュー」などの数字は、その数字が意味することを必ず人の行動・感情に翻訳してから書く
- お客さん目線とスタッフ目線を分けて整理し、求人LPでは「スタッフ目線」に変換する
